# Sigil Movement 使用文档

> 移动与运动动画**配套包** `com.likeon.gas.movement`。依赖 Sigil 核心包 `com.likeon.gas`（GitHub: [sigil-gas](https://github.com/forestlii/sigil-gas)）。命名空间不变（`Likeon.GAS`）。
>
> 设计定位：移动是 GameplayTag 状态总线的**消费方**，不属于能力系统本身——所以它单独成包。你可以只用核心配自己的移动，也可以装这个包获得开箱即用的移动 + 运动动画数据层。

## 1. 安装

本包依赖 Sigil 核心包，两者一起装：

1. 先装 `com.likeon.gas`（Sigil 核心）。
2. 再装 `com.likeon.gas.movement`（本包）。

（Package Manager → *Add package from disk…* → 分别选各自的 `package.json`。）

程序集：`Likeon.GAS.Movement`（Runtime，references `Likeon.GAS.Runtime`）+ `Likeon.GAS.Movement.Editor`。命名空间统一 `Likeon.GAS`，`using Likeon.GAS;` 即可。

## 2. 移动系统

`CharacterMovementSystemComponent`（需要 `CharacterController`）。数据驱动 + 状态总线对接 GAS。

```csharp
using Likeon.GAS;

var move = gameObject.AddComponent<CharacterMovementSystemComponent>();
// 同物体上有 ASC 时，Awake 自动把 ASC 设为标签 provider —— 移动状态会镜像到 ASC（可选集成）

move.PushAvailableMovementDefinition(movementDef);          // MovementDefinition 资产
move.SetMovementSet(GameplayTag.RequestTag("Movement.Set.Default"));
move.SetDesiredMovement(MovementTags.MovementState_Jog);    // Walk / Jog / Sprint
move.SetDesiredRotationMode(MovementTags.RotationMode_VelocityDirection); // 朝移动方向

// 每帧喂输入方向（世界空间）
move.SetInputDirection(camRelativeDir);
```

`MovementDefinition`（*Create → Likeon → GAS → Movement Definition*）：含若干 MovementSet，每个 Set 下有 Walk/Jog/Sprint 的 `MovementStateSetting`（Speed / Acceleration / BrakingDeceleration / RotationInterpolationSpeed…）。可栈式切换（如装备大剑 push 一套、卸下 pop）。

`MovementTags`：内置的移动相关 GameplayTag（`MovementState_Walk/Jog/Sprint`、`RotationMode_*`、`LocomotionMode_*` 等）。

## 3. 运动动画驱动

`LocomotionAnimationDriver`（挂角色，自动找 CharacterController / Animator / MovementSystemComponent）每帧从移动数据算出驱动值，既可直接读、也写进 Animator 参数：

```csharp
var driver = character.AddComponent<LocomotionAnimationDriver>();
// 直接读：driver.State.Speed / .CardinalDirection / .Lean / .Jumping / .ViewYawAngle …
// 或它写 Animator 参数（Speed / Direction / VelocityYaw / LeanForward / LeanRight /
//   Grounded / Jumping / Falling / ViewYaw / ViewPitch / PitchAmount）喂混合树。
```

产出涵盖：水平速率、相对朝向偏航、四/八向（带死区滞回防边界抖动）、倾身、空中态（跳跃顶点时间 / 下落时长 / 刚落地 / 着地预测）、视角相对朝向的 AimOffset、核心状态标签（LocomotionMode / RotationMode / MovementState / MovementSet）。

`LocomotionMath` 是一组纯函数（速度/偏航/方向/倾身等），可单独调用、易测试。

**示例 Controller**：菜单 *Likeon ▸ GAS ▸ Samples ▸ Create Locomotion Animator Controller* 一键生成一个对齐上述参数的分层 Animator Controller（Base 八向 2D 混合树 + Jump/Fall 状态 + 上半身 AimOffset 叠加层 + AvatarMask 占位），动画 clip 槽留空给你填。

> 说明：传统逐节点动画图方案在 Unity 无直接等价物，故本包用"Animator 数据驱动层 + 示例分层 Controller"替代；成品动画 clip 与最终手感由宿主工程提供。

## 4. 与 GAS 的关系（状态总线）

**状态总线**：调 `SetDesiredMovement(Sprint)` 后，`Movement.State.Sprint` 会自动镜像到 ASC 的松散标签——于是核心包[输入分发层](https://github.com/forestlii/sigil-gas)的"冲刺时一键滑铲、否则下蹲"这类**状态驱动按键多态**就接上了真实移动状态。

ASC 镜像是**可选**的：移动系统没有 ASC 也能跑（`SetGameplayTagsProvider` 不设即可），只是不会把状态推到技能层。

> 本包只依赖核心的 `GameplayTag` 词汇与（可选的）`AbilitySystemComponent`，不反向耦合——核心可独立使用。
